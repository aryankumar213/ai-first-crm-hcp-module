# AI-First CRM HCP Module: Log Interaction Screen

This project is a prototype of an **AI-First Customer Relationship Management (CRM)** system tailored for Healthcare Professionals (HCPs) and medical/pharmaceutical sales representatives. It focuses on the **Log Interaction Screen**, providing a dual-interface layout where representatives can log detailing activities using either a structured form or a conversational AI assistant. Both interfaces are synchronized in real-time.

## Features

- **Structured Input Form**: Comprehensive fields capturing HCP name (searchable autocomplete), interaction type (Meeting, Call, Email, Webcast), date/time, attendees, topics discussed, outcomes, and follow-up actions.
- **Conversational AI Assistant**: Chat interface driven by an agentic workflow that parses unstructured log inputs and dynamically fills out the left-hand form fields.
- **Multi-Model Agent Pipeline**: Powered by **LangGraph** and **Groq**'s LLMs:
  - **`llama-3.3-70b-versatile`** is utilized for both precise structured data extraction (Pydantic models) and friendly conversational chat generation.
  - Features a **local simulator fallback mode** that runs using heuristics if no Groq API Key is configured, allowing for complete offline testing.
- **Pharma-Specific Modules**:
  - **Materials Shared & Samples Distributed**: Manage and search trial packages (e.g. *OncoBoost 10mg*) and clinical materials (e.g. *OncoBoost Phase III PDF*) synced with a SQLite database.
  - **Compliance Consent Voice Notes**: Simulated speech-to-text recording that requires explicit user confirmation of HCP consent before capturing summaries.
  - **AI Suggested Follow-ups**: Proactive action suggestions generated by the LLM which populate the follow-up text area on click.

## Getting Started

### 1. Prerequisites
- **Node.js** (v18 or higher)
- **Python** (v3.10 or higher)
- **Groq API Key** (Generate one at [console.groq.com](https://console.groq.com/))

### 2. Configure Environment Variables
Navigate to the `backend/` folder. Copy the `.env.example` file to a new file named `.env`:
```bash
cp backend/.env.example backend/.env
```
Open the `.env` file and replace the placeholder value with your actual Groq API Key:
```env
GROQ_API_KEY=gsk_your_groq_api_key_here
```

### 3. Backend Setup
Navigate to the `backend/` directory, set up a virtual environment, install the dependencies, and start the FastAPI server:

```bash
cd backend
python -m venv venv

# Activate on Windows (CMD/PowerShell)
venv\Scripts\activate
# Activate on macOS/Linux
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Seed the database with mock life science data
python database.py

# Start the uvicorn development server
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```
The backend server will run at **`http://127.0.0.1:8000`**.

### 4. Frontend Setup
Open a new terminal tab, navigate to the `frontend/` directory, install packages, and start Vite:

```bash
cd frontend
npm install
npm run dev
```
The frontend web app will run at **`http://localhost:5173/`**. Open this URL in your browser to interact with the screen.
